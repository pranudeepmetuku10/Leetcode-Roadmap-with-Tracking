# MAANG (Big Tech)

## 🔵 Google

**Rounds:** 3-4 coding rounds (1-2 hours each)  
**Difficulty:** Hard focus (50% medium, 50% hard)  
**Style:** Pattern mastery + system thinking

### Top Patterns
1. **Graphs** (DFS/BFS emphasis)
2. **Dynamic Programming** (2D heavy)
3. **Binary Search** (optimization)
4. **Trie** (prefix/string problems)

### Must-Do Problems (by LC #)
- 200 (Number of Islands) — DFS/BFS classic
- 98 (Validate BST) — Tree property thinking
- 76 (Minimum Window) — Sliding window hard
- 208 (Implement Trie) — Interview favorite
- 207 (Course Schedule) — Topological sort
- 297 (Serialize/Deserialize) — Tree mastery

### Interview Style
- Very thorough problem exploration
- Tests collaborative debugging
- Expects clean, production-ready code
- Often has follow-ups (variant problems)

### Tips
- **Practice hard problems** — 50% of your time
- **Explain trade-offs** — Space vs time
- **Ask clarifying questions** — They reward this
- **Think about scaling** — How would this work at Google scale?
- **Use Google-style naming** — Clear, descriptive variables

### Red Flags (Don't!)
- ❌ Brute force without mentioning optimization
- ❌ Code that isn't production ready
- ❌ Not thinking about edge cases
- ❌ Overcomplicating simple problems

- ❌ Not thinking systematically
- ❌ Incorrect solution (even if attempt was good)
- ❌ Not double-checking your work

## 📋 Interview Preparation Checklist

**Before every interview:**

- [ ] Reviewed company's top patterns
- [ ] Solved their must-do problems
- [ ] Familiar with their interview style
- [ ] Known my weak patterns (practice extra)
- [ ] Have edge case strategy ready
- [ ] Understand my speaking pace (not too fast)
- [ ] Know my clarification process
- [ ] Can code without IDE

**Day before:**

- [ ] Sleep well (more important than last-minute cramming)
- [ ] Light review of patterns (no new problems)
- [ ] Relaxation routine
- [ ] Set up quiet interview space

**During interview:**

- [ ] Clarify problem before coding
- [ ] Talk while coding
- [ ] Test your solution
- [ ] Check edge cases
- [ ] State complexity analysis
- [ ] Be ready for follow-ups
